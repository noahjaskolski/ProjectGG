Welcome to Level 3

Within the Linux operating system, there are different types of users with varying levels of access to the files in the hierarchy. Each user is identified by a system UID (User Identification number) that is stored in /etc/passwd file. The hashed password for users are stored in /etc/shadow. In the previous lesson, we learned more about the file hierarchy and its purpose.

Let's take a look at this file. Login to the terminal. Enter command: cd /etc. Then enter: cat passwd. You can see the contents of the file of users along with each user's username, password, UID, GID (Group User Identification), GECOS (additional information of user if entered), absolute path of user's home directory and the login shell of the user.

Syntax:

[username]:[password]:[UID]:[GID]:[GECOS]:[home_dir]:[shell_path]

Two categories of users based on access are:

  • Superuser/root/administrator:this user has access to all files on the system.
  • Normal users: have limited access

Sudo is an important command that allows a permitted user to execute a command as the superuser. Operative word is permitted. Not all users will have this level of access. Let's try to view the /etc/shadow file with command: cat /etc/shadow

Permission was denied because only a superuser/root/admin can view this file. Try sudo cat /etc/shadow.

Success! you can now see the users.

Syntax:

[username]:[enc_pwd]:[last_pwd_change]:[pwd_validity]:[warn_date]:[acc_validity]:[acc_disablity]

We can see towards the bottom your user name vintage and the encrypted password following your username. This is a hashed password that cannot be decrypted. The beginning of this hash is denoted with $6 which represents a SHA512 hashing.
